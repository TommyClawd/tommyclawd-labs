Methodology

How the ROI score works

This dashboard uses the score already calculated in the dataset generator. In plain English: bigger average payouts help, better response efficiency helps, steadier recent activity helps, and heavy researcher crowding hurts.

ROI score = (average bounty × response efficiency × activity factor) ÷ √participants

activity factor = clamp(resolved reports ÷ max(1, reports in last 90 days × 4), 0.5, 2.0)

The square-root crowding penalty matters: a program does not need to be obscure to rank well, but it does need enough payout quality and throughput to justify how many hunters are already there.

Reading the table

What green, yellow, and red mean

  • High ROI = top third of scores in this dataset.
  • Medium ROI = middle third.
  • Low ROI = bottom third, including zero-score programs.
  • Search narrows by program name or handle; sorting lets you compare any field directly.
Caveats

Use this as a screener, not a guarantee

  • The score is a heuristic, not a true expected-value model.
  • It does not capture severity mix, duplicate rate, asset sprawl, or researcher skill fit.
  • Programs with tiny sample sizes can look amazing; programs with huge crowds can still be worth it if you have a real edge.
